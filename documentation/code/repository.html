Repository usage

GIT pull/update

When doing an update watch for changes in the following files:

  • code/config/*
  • code/htdocs/config2/settings-dist.inc.php
  • code/htdocs/doc/sql/*
  • code/htdocs/lib/settings-dist.inc.php
  • code/htdocs/documentation/*
In most cases you have to review the changes and update your configuration or database. The update is done manually.
  • Changes in the setting files may have to be modified in the local config file.
  • Changes in SQL table definitions have to be installed in the database via phpmyadmin. If there is a new table SQL file, you can create the new column or modify the existing column.

    If data.sql changes, you have to execute it with mysql binary 
    cat data.sql | mysql -uroot -p <database>
    If maintain.php changes, you have to execute it with your webbrowser.

  • Changes in the documentation should be reviewed, of course.

GIT push

Commit your changes to an development branch when you have finished the developlment.
When these changes have been tested, they will be merged into the origin.
